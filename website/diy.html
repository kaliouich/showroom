Do It Yourself

Build This Infrastructure

A step-by-step technical guide with the exact commands needed to replicate this entire DevOps ecosystem on an Oracle Cloud Always Free instance.

☁️

1. Provision Oracle Cloud VM

▼

Create an ARM Ampere A1 instance (4 OCPUs, 24GB RAM) on Oracle Cloud Always Free tier using Ubuntu 24.04.

# SSH into your newly created instance
ssh -i /path/to/private/key ubuntu@YOUR_PUBLIC_IP
🛡️

2. Configure Firewall (iptables)

▼

Oracle Cloud Ubuntu images come with strict iptables rules. We need to explicitly open HTTP (80) and HTTPS (443) ports.

# Insert rules to allow ports 80 and 443
sudo iptables -I INPUT 6 -m state --state NEW -p tcp --dport 80 -j ACCEPT
sudo iptables -I INPUT 6 -m state --state NEW -p tcp --dport 443 -j ACCEPT

# Save the rules permanently
sudo netfilter-persistent save
🚢

3. Install K3s (Lightweight Kubernetes)

▼

Deploy a single-node Kubernetes cluster without the default Traefik ingress controller to keep it clean. This command installs the K3s server.

curl -sfL https://get.k3s.io | INSTALL_K3S_EXEC="server --disable traefik" sh -
            
# Verify the node is ready
sudo k3s kubectl get node
🐙

4. Install ArgoCD (GitOps)

▼

Install ArgoCD into the cluster to manage its state declaratively from a Git repository.

sudo k3s kubectl create namespace argocd
sudo k3s kubectl apply -n argocd -f https://raw.githubusercontent.com/argoproj/argo-cd/stable/manifests/install.yaml

# Wait for ArgoCD pods to be ready
sudo k3s kubectl get pods -n argocd -w
🔑

5. Port-Forward ArgoCD UI

▼

Retrieve the initial admin password generated by ArgoCD and port-forward the server to access the UI locally.

# Retrieve the initial admin password
sudo k3s kubectl -n argocd get secret argocd-initial-admin-secret -o jsonpath="{.data.password}" | base64 -d; echo

# Port-forward the ArgoCD UI to access it
sudo k3s kubectl port-forward svc/argocd-server -n argocd 8080:443
🚀

6. Bootstrap "App of Apps"

▼

Apply your root ArgoCD Application that points to your Git repository containing the rest of the cluster configuration (Prometheus, Demo App, etc.).

# Create a root-app.yaml file
apiVersion: argoproj.io/v1alpha1
kind: Application
metadata:
  name: root-app
  namespace: argocd
spec:
  project: default
  source:
    repoURL: 'https://github.com/YOUR_GITHUB/showcase-infra.git'
    targetRevision: HEAD
    path: apps
  destination:
    server: 'https://kubernetes.default.svc'
    namespace: argocd
  syncPolicy:
    automated:
      prune: true
      selfHeal: true

# Apply it
sudo k3s kubectl apply -f root-app.yaml
