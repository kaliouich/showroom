Troubleshooting

Technical Issues Resolved

A deep dive into the engineering challenges faced and overcome while deploying this architecture on Oracle Cloud.

🕸️

CI/CD Runner Network Isolation

Issue: Gitea Actions containers suffered from MTU fragmentation and NAT translation failures when communicating with K3s Pod IPs.

Resolution: Forced CI job containers to run on the host's network namespace (container.network: host), allowing seamless internal DNS resolution without NAT overhead.

🔐

GitOps Push Authentication

Issue: Deployment pipeline failed to commit updated K8s manifests back to Gitea due to standard GitHub/Gitea token context limitations.

Resolution: Injected a dedicated access token directly into the remote URL to bypass the context boundaries for automated GitOps pushes.

⚡

ArgoCD gRPC Interference

Issue: ArgoCD became completely inaccessible (502 Bad Gateway) as the Linkerd Service Mesh intercepted and broke internal gRPC TLS handshakes.

Resolution: Selectively disabled Linkerd proxy injection on the argocd namespace and rolled out new pods to restore internal control plane communication.

🐳

ImagePullPolicy Stale Caching

Issue: Kubernetes worker nodes (containerd) failed to pull or cache-invalidated images due to internal headless service routing.

Resolution: Switched registry target to the external proxy domain and set imagePullPolicy: Always to force strict layer hash validation.

🚀

Node.js vs Nginx Port Bindings

Issue: A frontend CSS update accidentally reverted the container build to an old Nginx Dockerfile, causing a port mismatch (80 vs 3000) for the live metrics backend.

Resolution: Restored the server.js Node.js proxy architecture, rebuilt the image via nerdctl, and executed a rolling K8s deployment update.

🔑

ArgoCD RBAC Credentials

Issue: The Read-Only Guest visitor account was rejected by ArgoCD despite proper RBAC mapping in the ConfigMap.

Resolution: Generated a raw bcrypt hash manually via Python and directly patched the argocd-secret to inject the guest password securely.
